## Overview

McCartney provides a centralized space to organize, manage, and collaborate on your project documentation. You create structured docs with MDX support, version history, and search. Start by inviting team members and uploading your first Markdown file.

<Callout kind="info">
  McCartney uses brand color `#3B82F6` for primary elements. Apply it in custom themes via CSS variables.
</Callout>

## Key Features

Explore core capabilities that streamline your documentation workflow.

<Columns cols={3}>
  <Card title="MDX Support" icon="code">
    Write rich docs with components like `<Callout>` and `<Steps>`. Embed interactive elements directly.
  </Card>
  <Card title="Version Control" icon="git-branch">
    Track changes with Git integration. Roll back to any version effortlessly.

  <Card title="Search & Navigation" icon="search">

## Benefits for Project Teams

You collaborate without version conflicts. Developers document APIs using `<ParamField>` components, while designers add screenshots. Teams stay aligned with real-time previews and comments.

- **Centralized Access**: One link shares everything.
- **Customizable**: Tailor layouts with YAML frontmatter.
- **Scalable**: Handles large projects with thousands of pages.

## Quick Start

Get up and running in minutes.

<Steps>
  <Step title="Create Account" icon="user-plus">
    Visit `https://mccartney.app` and sign up with your email.
  </Step>
  <Step title="New Space" icon="plus">
    Click **New Space** and name it for your project.
  </Step>
  <Step title="Upload Docs">
````markdown
# My First Page

Welcome to McCartney!
````

    Drag and drop Markdown or MDX files.
  </Step>
  <Step title="Invite Team" icon="users">
    Share the space link and add collaborators.
  </Step>
</Steps>

## Brand Guidelines

Follow these to maintain consistency.

<Tabs>
  <Tab title="Colors" icon="palette">
    Primary: `#3B82F6`  
    Use for buttons and links.  
    Secondary: `#1E40AF`
  </Tab>
  <Tab title="Typography" icon="font">
    Headings: Inter font, bold.  
    Body: 16px base size.
  </Tab>
  <Tab title="Components" icon="components">
    Stick to available components like `<Card>` and `<Steps>`. Avoid custom JSX.
  </Tab>
</Tabs>
